12 Content Integrity Set (CIS)

  • Context of use: One or more malicious peers want to poison the stream.
  • Clues: The splitter should identify them and stop sending more chunks.
    1. Security level 1: Trusted peers send to the splitter a hash of each chunk and the splitter checks if the chunks have been poisoned.
    2. Security level 2: Chunks are digitally signed (to allow peers to know if their are been attacked) and the splitter gathers statistics from all peers of the team.
